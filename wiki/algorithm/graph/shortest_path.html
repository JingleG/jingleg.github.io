Shortest Path

Dijkstra

基于贪心，每次选取最近的点加入集合

不能存在负权边，无需考虑是否自环，重边取min即可

朴素 $\mathcal{O}( n^2 )$

堆优化 $\mathcal{O}( m \log{n} )$

简化每次搜寻最近点，使用堆存储当前所有的点，使用优先队列每次加入新更新的点（堆大小为m）或手写堆更改堆中任意元素（堆大小为n），实际$m <= n^2, \log m <= 2\log n$, 无差别

Bellman-ford

  • 迭代第n次代表经过n个路径的最短路，需要使用bak防止单次迭代更新多次路径
  • 如果第n次依然发生修改说明存在负权回路（最多n个点，重复访问）
  • 基于dp，只要最短路中不含有负环即可
  • $\mathcal{O}( mn )$

经过n条路径的最短路

SPFA

Shortest Path Faster Algorithm 队列优化版Belman-ford，一般 $\mathcal{O}( m )$, 最差 $\mathcal{O}( mn )$

图中必须不含有负环, 针对bellman-ford算法优化dist[a]未更新时不修改dist[b]，使用队列存储所有优化过的节点进行循环

判断负环

使用cnt[i]标记从起点到i点的最短路径所经过的边数，每次更新时使用中间点更新cnt[e] = cnt[v]+1，若从起点到当前点i所经过的路径大于等于n, 则说明至少经过了n+1个点，存在负环。Drawer principle

  • 注意起始时需要将所有点加入队列，需要求任意起点的负权回路

Floyd

多源路径，可有负权边，无负权回路，一般使用邻接矩阵，本质为动态规划，背包优化后为d[i,j] = min(d[i,j], d[i,k] + d[k,j]) 时间 n^3

d[k,i,j]表示从第i个点，经过1 ~ k个中间点，到达j的最短距离
